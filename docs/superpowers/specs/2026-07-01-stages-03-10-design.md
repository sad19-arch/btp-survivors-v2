# Design — Stages 03-10 (cycle de chantier complet)

## Contexte

`main` contient un jeu jouable de bout en bout avec **2 stages** en DA 16-bit
(terrain vierge, terrassement), un **système multi-stage** (`?level=<phase>` →
phase + assets via `src/render/stages.ts`), l'équilibrage « skill récompensé »
tuné (harnais `npm run sim:check`, marge zéro) et les correctifs de dettes.

But : dérouler les **8 stages restants** (fondations → livraison/audit) pour
compléter la colonne vertébrale du chantier, **sans casser l'équilibrage**.

## Principes (validés)

1. **Re-skin à stats identiques.** Chaque stage a 3 ennemis (base/fast/tank) =
   NOUVEAUX ids de contenu aux **stats copiées** de paperasse/inspecteur/huissier.
   La sim se comporte à l'identique → `sim:check` reste VERT **par construction**.
2. **Boss = créature unique par stage** (jamais un engin). La sim spawne toujours
   le mini-boss `contremaitre` (mêmes stats) au climax 5:00 ; **seul le SKIN change
   par stage** (côté rendu).
3. **Engins/véhicules = habillage** (props/landmarks), adaptés à la phase, riches
   (grues, pelleteuses, bétonnières, rouleau compresseur…). Réutilisables entre
   phases quand c'est cohérent.
4. **DA stricte**, pipeline PixelLab prouvé, golden-batch/stage-par-stage.

## Le seul changement de code : boss-skin par stage

Aujourd'hui `GameScene` mappe `isBoss → 'boss'` (feuille partagée `ground_keeper`).
On rend le boss **stage-aware** comme les ennemis :

- `src/render/stages.ts` : `StageRender` gagne un champ `boss: StageEnemySprite`.
- `GameScene.syncSprites` : pour `en.isBoss`, utiliser `this.stage.boss` (clé +
  échelle) au lieu de la clé partagée `'boss'`. Repli cercle si texture absente.
- `preload` : charger `this.stage.boss` (au lieu de/à côté de `SHARED_SHEETS.boss`).
- Stage 01 : `boss` = ground_keeper (existant). Stages 02-10 : skin propre.

`src/core` **inchangé** pour le boss (toujours `contremaitre`, mêmes stats).

## Contenu (data-driven, `src/content`)

- `enemies.ts` : +24 ennemis (3 × 8 stages), stats **copiées** des archétypes
  stage 01 (base=paperasse, fast=inspecteur, tank=huissier). Noms thématiques.
- `phases.ts` : définir les 8 phases (03-10) avec `enemyPools {base,fast,tank}`
  pointant vers ces ids. `order` 3..10. `accentColor` par phase.
- Pas de nouveau boss côté contenu (skin géré au rendu). `MINI_BOSS_ID` inchangé.
- Validation boot existante (`phasePoolIds`) : les ids référencés existent.

## Recette de production par stage (réutilise le pipeline prouvé, résumable)

1. **Sol** : `create_tiles_pro` (top-down plat, segmentation) → `flatten-tiles.mjs`
   (normalise/anti-damier) → `public/stageNN/ground/tile_0..5.png`.
2. **Décalques** (2-3) : `create_map_object` plats → trim → `public/stageNN/decals/`.
3. **Ennemis** (3) + **boss** (1) : `create_character` v3 → `animate_character`
   walk 4-dir → download → `pack-character.mjs` (cellule 256) →
   `public/stageNN/enemies/` + `public/stageNN/boss/`.
4. **Engins/props** : `create_map_object` → trim → `public/stageNN/props/`.
5. **Intégration** : entrées `stages.ts` (ground/decals/props/enemies/boss) +
   phase dans `phases.ts` + 3 ennemis dans `enemies.ts`.
6. **Calibrage** : `measure-sprite-size.mjs` → échelles (cibles ~ base 73 /
   fast 69 / tank 86 / boss 144 px affichés).
7. **Validation** : capture `?level=<phase>` + `sim:check` VERT + type-check + lint
   + Vitest.

## Tests / garde-fous

- **Équilibrage intact** (re-skins à stats identiques + boss identique) →
  `sim:check` VERTES à chaque stage. C'est l'invariant clé.
- **Vitest** : les pools de phase pointent vers des ids d'ennemis valides.
- **1 capture en jeu par stage** (ennemis skinnés + boss + engins + sol).
- **type-check + lint** verts ; pas de mélange DA.
- `src/core` quasi intouché (contenu +24 ennemis, aucune logique) ; sim `sim:check`
  identique au baseline.

## Portée & découpage (multi-sessions)

- **Ce spec + le 1er plan** couvrent : le changement de code boss-skin +
  le scaffolding contenu (8 phases + 24 ennemis + entrées `stages.ts`) +
  l'enrichissement stages 01-02 + la production du **stage 03** (pilote de la
  recette « stage + boss-skin »).
- Les **stages 04-10** suivent la même recette (production résumable, un stage
  par vague), validés stage par stage. La génération d'assets (~10-15/stage) est
  le long pôle → étalé sur plusieurs sessions.
