## Task 1 — Fraîcheur de l'overlay (fix régression B2 Task 5) + e2e au vert

**Files :**
- Modify: `src/app/app.ts` (bump du compteur `frame` sur chaque mutation d'état)
- Test: `tests/unit/getStateCache.test.ts` (cas mutation-sans-advanceTime)
- Modify: `tests/e2e/seam.spec.ts` (l.130 : `toBe(3)` → `toBe(4)`)

**Contexte :** B2 Task 5 a ajouté un cache par frame : l'overlay lit `app.getStateForFrame(app.frameId)`. Le compteur privé `App.frame` n'est incrémenté que dans `advanceTime`. En `testMode`, `GameScene.update` n'appelle pas `advanceTime`, donc `nav()`/`pause()` seuls ne bumpent pas `frame` → l'overlay renvoie l'objet caché périmé (focus figé, Pause absent). En jeu réel c'est masqué car `advanceTime` tourne chaque frame.

**Correctif :** rendre le bump systématique. Ajouter `private bumpState(): void { this.frame++ }` et l'appeler en tête de **chaque méthode publique mutatrice** de `App` : `nav`, `confirm`, `back`, `togglePause`/`pause`/`resume`, `start`, `restart`, `clickItem`, `chooseUpgrade`, ouverture/fermeture Options, sélection de stage, + helpers debug (`debugGrant`/`debugAddXp`/`debugSpawnBoss`/`debugSpawnChestOnPlayer`/`debugSpawnEnemies`). `advanceTime` bump inchangé. `getState()` reste frais/non-caché. Contrat auditable : « toute méthode publique qui mute l'état bump `frame` ».

- [ ] Step 1 — Test qui échoue (`getStateCache.test.ts`) : construire l'`App` comme dans `tests/unit/app.test.ts` ; `const f0 = app.frameId ; app.nav('down')` puis `app.getStateForFrame(app.frameId)` doit refléter le nouvel index de focus (≠ celui avant `nav`), et `app.frameId !== f0`. Deuxième cas : après `app.pause()` (partie lancée), `getStateForFrame(app.frameId).screen === 'paused'`.
- [ ] Step 2 — Lancer, vérifier l'échec (méthode ne bump pas encore).
- [ ] Step 3 — Implémenter `bumpState()` + appels en tête des mutateurs.
- [ ] Step 4 — `tests/e2e/seam.spec.ts:130` : `toBe(3)` → `toBe(4)`.
- [ ] Step 5 — Gates : type-check + lint + `npx vitest run` + **`npx playwright test` → 0 échec** (title-focus `screens.spec.ts:18` et Pause `:33` doivent passer une fois l'overlay rafraîchi). Si un échec title/pause persiste → investiguer (NE PAS masquer via l'assertion). Commit.
