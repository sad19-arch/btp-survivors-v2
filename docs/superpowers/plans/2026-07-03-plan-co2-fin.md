## Task 2 — Revive (à terre → coéquipier relève en maintenant l'action)

**Files :** `src/input/intents.ts`+`gamepad.ts`+`keyboard.ts`+`players.ts` (entrée `action`), `src/core/types.ts` (`PlayerInput.action?`, composant `revive{progress}`, `PlayerState.downed`+`reviveProgress`), `src/core/systems/revive.ts` (nouveau), `src/core/simulation.ts` (`applyPlayerInputs`/`step`/`collectPlayers`), `src/content/config.ts` (`REVIVE`), `src/render/scenes/GameScene.ts` (rendu à terre + barre), tests.

### 2.1 Entrée `action` (maintien)
- `FrameInput` gagne `action: boolean`. `GamepadInput` : `action = down(BTN.A)`. `KeyboardInput` : `action =` touche `E` maintenue. `mergeFrames` : `action = a.action || b.action`. `routeInput` : `app.setInput(id, { move, attack:false, action: frame.action })`.
- `PlayerInput` : `action?: boolean` **OPTIONNEL** (ne casse pas les `setInput({move,attack})` existants ; lu `?? false`). `attack` reste inutilisé.

### 2.2 `reviveSystem` (core pur)
- `REVIVE = { radius: 80, fillSeconds: 3, decaySeconds: 2, hpFraction: 0.5 }`.
- `reviveSystem(world, inputs: Map<number,PlayerInput>, dtMs)` : pour chaque joueur `hp<=0` → composant `revive{progress:0}` si absent ; chercher un joueur **vivant** (`hp>0`) à `<=radius` **tenant `action`** (`inputs.get(sonId)?.action`) ; si ≥1 → `progress += dtMs/1000/fillSeconds`, sinon `progress -= dtMs/1000/decaySeconds` (clamp ≥0) ; `progress>=1` → `hp = maxHp*hpFraction` + retirer `revive`. Joueur `hp>0` → retirer `revive` résiduel. **Gated par la proximité du bon releveur.** Pur/déterministe.
- `simulation.step` : appeler `reviveSystem` **après collision/reap**, **avant `updateGameOver`**. Solo : joueur à terre → 0 releveur → jamais relevé → game-over identique → **sim:check diff 0**.

### 2.3 View-state + rendu
- `collectPlayers` : `downed = hp<=0` (partie en cours) ; `reviveProgress` = progrès du composant `revive` (0 sinon). Additif.
- `GameScene.syncSprites` : joueur `downed` (pas game-over) → **visible, teinté gris / couché** (au lieu d'invisible) + **barre de revive** au-dessus (couleur `PLAYER_COLORS[id]`, remplie par `reviveProgress`) ; Graphics géré comme les anneaux (clear/redraw, pas de leak).

**Tests :** revive (releveur à portée tenant action → progresse → PV=50% ; hors portée/relâché → décroît ; solo → jamais relevé ; dernier à terre → game-over) ; `action` propagé (players/routeInput). **Gate :** `npm run type-check` (sortie) + lint + `npx vitest run` + `npx playwright test` (solo inchangé) + `npm run sim:check` **diff 0**.
