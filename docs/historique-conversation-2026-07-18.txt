PARTIE 1 — CONTEXTE DU PROJET (résumé du travail antérieur, 30/06 → 17/07)
========================================================================

BTP Carnage est un jeu web type Vampire Survivors, univers chantier BTP,
reconstruction propre d'un prototype précédent (design bon, code mauvais).
Règles d'architecture strictes (voir CLAUDE.md) : séparation sim/rendu,
déterminisme (RNG seedé, FixedClock), ECS-lite data-oriented, N-joueurs
prêt dès le départ, data-driven, typage strict, UI componentisée, contrôle
100% manette+clavier, DA 16-bit stricte via palette unique.

Grandes étapes déjà livrées avant cette session (résumé) :
- Boucle de jeu complète (titre → game over), 3 armes, mini-boss, 10 stages
  avec identité visuelle propre (landmarks, PNJ métier, sols dédiés).
- Système d'armes façon Vampire Survivors : niveaux, passifs, cartes,
  évolutions via coffres.
- Coopération 2-4 joueurs, caméra centroïde, revive, scaling difficulté.
- IA ennemis à 5 comportements + directeur de vagues chorégraphié.
- Monde ×10 par streaming de décor en chunks.
- Mode Carnage secret (Konami), destructibles, économie de coffres/casino.
- Refonte arcade (P0-P7) : logo, écrans, mobile.
- Stage Composer Editor (éditeur de niveaux in-game) avec multi-sélection,
  copier/coller, palette d'assets.
- Pipeline d'assets PixelLab (golden batch → production stage par stage)
  et audio (musiques, SFX zzfx puis ElevenLabs, ducking voix).
- ~1600-2300 tests Vitest, ~175-185 tests Playwright, harness de
  simulation headless déterministe (`npm run sim` / `sim:check`).

Un fait central acquis pendant cette période : mes briefs de contrôleur
se sont trompés à de très nombreuses reprises (23 fois recensées à un
moment donné) — presque toujours rattrapés par des agents dispatchés à
qui j'avais explicitement demandé de « signaler tout écart, ne pas
appliquer mécaniquement ». Cette consigne est devenue le principe
directeur de toute la méthode de travail.

État au moment de la reprise : branche `feat/editor-palette`, ~75 commits
d'avance sur `main`, arbre en cours de travail par plusieurs agents en
parallèle (chantier ouvert sur les engins de chantier, les PNJ métier,
le camion, l'audio, les cinématiques).
